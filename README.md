## Question 2: HTML Elements

### 1. Most Challenging Elements
The following elements were challenging to implement:

- `<table>` – Structuring rows, headers, and body correctly  
- `<form>` – Required combining multiple input elements  
- `<fieldset>` – Understanding how to group form elements  
- `<textarea>` – Managing size with rows and columns  
- `<nav>` – Structuring navigation links properly  

### 2. Use of Semantic Elements
I used semantic elements to structure the website:

- `<header>` – Contains the title and navigation  
- `<main>` – Holds all main content  
- `<section>` – Divides content into meaningful parts  
- `<article>` – Represents individual fruit content  
- `<footer>` – Contains copyright and external links  

This improves readability and accessibility.

### 3. Most Useful Element
The `<section>` element was the most useful because it helped organize content into clear and logical parts.

## Question 3: HTML Attributes

### 1. Essential Attributes
- `href` – Used for navigation links  
- `src` – Used to display images  
- `type` – Defines input types in forms  

### 2. Class vs ID Usage
- `id` – Used for unique elements (e.g., navigation targets like `#home`)  
- `class` – Used for grouping elements (e.g., styling text like "intro")  

### 3. Attribute Improving UX
The `placeholder` attribute improved user experience by guiding users on what to input in form fields.
